Базовое использование:
import os
from anthropic import Anthropic

client = Anthropic(
    api_key=os.environ.get("ANTHROPIC_API_KEY"),  # This is the default and can be omitted
)

message = client.messages.create(
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": "Hello, Claude",
        }
    ],
    model="claude-sonnet-4-5-20250929", # это будет базовая модель в приложении
)
print(message.content)

асинхронное использование
import os
import asyncio
from anthropic import AsyncAnthropic

client = AsyncAnthropic(
    api_key=os.environ.get("ANTHROPIC_API_KEY"),  # This is the default and can be omitted
)


async def main() -> None:
    message = await client.messages.create(
        max_tokens=1024,
        messages=[
            {
                "role": "user",
                "content": "Hello, Claude",
            }
        ],
        model="claude-sonnet-4-5-20250929",
    )
    print(message.content)


asyncio.run(main())

Б. Логика перевода "По файлам" (File-based / Batch):
Сейчас перевод идет построчно. Нужно добавить режим BatchTranslation:

    Парсинг: Читаем .yml файл. Разбиваем его на чанки (группы) по 50-100 ключей (настраиваемый параметр), чтобы не перегрузить контекст модели.

    Промптинг: Формируем системный промт: Есть уже стандартный в promts.py но улучши его чтобы ии не оставлял MD обертку и писал обычным текстом тоесть чтобы можно сказать программа принимала ответ с ключами и их перерводом

    Обработка ответа: Получаем ответ модели, парсим его и собираем обратно в структуру файла.

    Валидация целостности: Проверяем, что количество ключей на выходе совпадает с количеством на входе. Если нет — помечаем чанк как ошибочный

2. GUI: Редизайн и Таблица проверки (Review Mode)

А. Библиотека стилей:
Использовать библиотеку PyQt6-Fluent-Widgets (pip install "PyQt6-Fluent-Widgets[full]" -i https://pypi.org/simple/), чтобы уйти от стандартного вида Windows 95. Интерфейс должен быть современным (Dark Mode).

Б. Окно "Таблица сравнения" (Post-Translation Review):
После завершения перевода открывается новое окно (или вкладка) с таблицей (QTableWidget):

    Колонки: "Ключ", "Оригинал", "Перевод", "Статус".

    Функционал:

        Возможность редактировать ячейку "Перевод" вручную.

        Чекбокс выбора строки для повторного перевода.

        Кнопка "Перевести выбранные" (отправляет запрос только по отмеченным строкам).

        Кнопка "Сохранить файл" (записывает изменения на диск).

        Навигация: Кнопки "Предыдущий файл" / "Следующий файл".

Асинсихнронность интерфейса и прогресс бары есть тебе остается их портировать чуток

Вместо стандартных виджетов PyQt6 используй их современные аналоги:

    Главное окно должно наследоваться от FluentWindow (а не QMainWindow).

    Вместо QPushButton используй PushButton или PrimaryPushButton (для главных действий).

    Вместо QLineEdit используй LineEdit.

    Вместо QTableWidget используй TableWidget (он красивее и поддерживает закругления).

    Для навигации (меню слева) используй встроенный в FluentWindow механизм NavigationInterface. Сделай три страницы (SubInterface):

        Home: Выбор файлов и настроек.

        Process: Лог перевода / Прогресс бар.

        Review: Таблица сравнения результатов.

    Обязательно включи поддержку темной темы: setTheme(Theme.DARK).
